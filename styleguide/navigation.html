Navigation

Contrary to popular belief the navigation element doesn't need to contain a list. (N.B. The HTML empty comments in-between nav items are to combat the inline-block whitespace issue).

Nav item oneSecond itemNav item 3Last item 
<nav role="navigation">
  <a href="#">Nav item one</a><!--
  --><a href="#">Second item</a><!--
  --><a href="#">Nav item 3</a><!--
  --><a href="#">Last item</a>
</nav>
Nav item one Second item Nav item 3 Last item 
<nav role="navigation" class="nav-vertical">
  <a href="#">Nav item one</a>
  <a href="#">Second item</a>
  <a href="#">Nav item 3</a>
  <a href="#">Last item</a>
</nav>

You can of course use a list, nothing is stopping you. It's super helpful for nested navigations. Note the .nav-sub class for these.

There are also some optional add ons here. Note the button to hide show the nav on small screens. Also there's a .nav-block class added for an alternative type of styling.
